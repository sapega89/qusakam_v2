# Інструкції для очищення кешу Godot Editor

## Проблема
Помилка "Parent path has vanished" або "Node was modified from inside an instance, but it has vanished" виникає через те, що Godot Editor зберігає кешовані дані про модифікації вузлів у папці `.godot`.

## Рішення

### Варіант 1: Використання скрипта (рекомендовано)

1. **Закрийте Godot Editor** (якщо відкритий)
2. Запустіть `clear_godot_cache.bat` в корені проекту
3. Відкрийте Godot Editor знову
4. Запустіть гру

### Варіант 2: Вручну

1. **Закрийте Godot Editor** (якщо відкритий)
2. Відкрийте папку проекту в Провіднику Windows
3. Увімкніть показ прихованих файлів:
   - Натисніть `Alt` → `View` → `Show` → `Hidden items`
   - Або в налаштуваннях папки: `View` → `Show` → `Hidden items`
4. Знайдіть папку `.godot` в корені проекту
5. Видаліть папку `.godot` (можна просто видалити або перейменувати)
6. Відкрийте Godot Editor знову
7. Запустіть гру

### Варіант 3: Через PowerShell (для користувачів PowerShell)

```powershell
# Закрийте Godot Editor перед виконанням
Remove-Item -Path ".godot" -Recurse -Force -ErrorAction SilentlyContinue
Write-Host "Кеш Godot Editor очищено!"
```

## Що робить очищення кешу?

Коли ви видаляєте папку `.godot`, Godot Editor:
- Перезавантажить всі сцени з новими UID
- Очистить кешовані дані про модифікації вузлів
- Очистить кешовані дані про стару структуру сцен
- Перезапустить файлову систему проекту

## Важливо

- **Закрийте Godot Editor перед видаленням `.godot`**, інакше можуть виникнути проблеми
- Після видалення `.godot` Godot Editor може трохи довше відкриватися при першому запуску (він перезавантажує всі ресурси)
- Всі зміни в сценах будуть збережені - видаляється тільки кеш, не самі файли

## Після очищення кешу

Після очищення кешу помилка "Parent path has vanished" має зникнути, оскільки:
- Видалено всі `editable path` для інстансованих сцен
- Змінено UID для всіх пов'язаних файлів
- Видалено посилання на неіснуючий `CentrallMenu` з коду
- Очищено кешовані дані про стару структуру
